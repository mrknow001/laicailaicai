# /src

用途：初始化正式 SRC / 企业渗透测试项目。

执行规则：

1. 先读取 `CLAUDE.md` 和 `.claude/skills/project-declaration/SKILL.md`。
2. 如果用户没有提供完整信息，询问以下内容：
   - 项目名称
   - 测试账号
   - 允许域名
   - 允许 IP / IP 段
   - 禁止域名
   - 禁止 IP / IP 段
   - 漏洞收录要求，未提供则使用默认
   - 注意事项，未提供则使用默认
3. 复制 `projects/_template` 整个目录到 `projects/<项目标识>/`，保持项目文件结构统一。
4. 按用户输入填写 `projects/<项目标识>/project.md`，并同步更新 `scope.md` 的白名单和黑名单摘要。
5. 项目标识使用项目名称的英文、拼音或用户指定短名；不确定时询问用户。
6. 创建完成前不得访问、扫描、fuzz 或测试目标。

输出：

- 当前项目路径
- 已记录的允许范围和禁止范围
- 使用的漏洞收录要求
- 使用的注意事项
- 下一步建议只围绕线索收集或策略讨论展开
