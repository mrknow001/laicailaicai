# /exp

用途：**人工补充 / 整理**经验库。自动沉淀（见 `CLAUDE.md`「经验自沉淀与召回常驻规则」）是主路径；`/exp` 只在你想手动记一条、补全遗漏或回顾整理经验库时用。

执行规则：

1. 先读取 `CLAUDE.md` 和 `.claude/skills/experience-manager/SKILL.md`，按其中的格式与够格标准执行。
2. 只沉淀方法论、判断模式、指纹关联、避坑经验；满足「可复用 + 非显然 + 可浓缩」三标准，不够格不写。
3. 不保存敏感目标数据、真实用户数据、Token、Cookie、密钥、完整响应包。
4. 精炼优先，言多必失：经验抽象为可复用模式，不写项目特定的敏感细节或大段背景。
5. 写入 `memory/<name>.md`（一事一文件，带 frontmatter），并更新 `memory/INDEX.md` 索引；已有同主题则更新正文并刷新 `last-verified`。

输出：

- 写入 / 更新的文件与 `INDEX.md` 条目
- 经验摘要
- 适用场景
- 避免误用的边界
