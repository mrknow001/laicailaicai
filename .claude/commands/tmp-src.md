# /tmp-src

用途：创建临时渗透测试项目声明，用于用户暂时没有完整 SRC 项目信息但需要先整理规则的场景。

执行规则：

1. 先读取 `CLAUDE.md` 和 `.claude/skills/project-declaration/SKILL.md`。
2. 临时项目仍必须至少包含：
   - 项目名称
   - 允许域名或允许 IP
   - 禁止域名或禁止 IP，可为空但必须显式记录
3. 临时项目命名建议为 `<主域名>-<日期>`。
4. 创建时复制 `projects/_template` 整个目录到 `projects/<临时项目标识>/`，再填写 `project.md`。
5. 缺少允许范围时，只能整理声明，不能测试目标。
6. 默认漏洞收录要求、默认注意事项、默认补充说明必须写入项目文件。

输出：

- 临时项目路径
- 当前可工作的范围
- 仍需用户补充的信息
