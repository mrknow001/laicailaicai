# /update-note

用途：向当前项目追加补充说明，用于纠偏智能体行为或增加项目级临时规则。

执行规则：

1. 先定位当前项目的 `project.md`。
2. 将用户提供的内容追加到“补充说明”区域，保留时间和来源。
3. 补充说明可以加强约束、调整测试重点、降低风险。
4. 补充说明不能绕过全局安全红线；如果用户追加内容与安全红线冲突，必须拒绝执行冲突部分并说明原因。
5. 追加后，后续所有线索收集、漏洞假设和验证都必须读取并遵守补充说明。

适合追加的例子：

- 本项目不要再测试 URL 跳转和普通 XSS，集中测试未授权、越权和敏感信息泄露。
- 本项目禁止任何登录口测试，包括弱口令和验证码逻辑。
- fuzz 只能使用小字典，必须低速，并在出现限流时停止。
