# /generate-dict

用途：根据当前项目的业务、接口、参数、命名习惯生成低风险字典。

执行规则：

1. 先读取 `CLAUDE.md` 和 `.claude/skills/dict-generator/SKILL.md`。
2. 字典生成必须服务于当前渗透测试目标。
3. 优先生成：
   - 路径
   - 参数名
   - 备份文件名
   - 业务关键字
   - 排序字段
4. 用户名、密码类字典只能用于风险研判和报告说明，禁止用于撞库、密码喷洒、真实账号爆破。
5. 输出必须包含适用目标、生成依据、风险等级和建议请求量。

禁止：

- 生成用于真实账号爆破的密码喷洒计划。
- 生成破坏性 payload 字典。
- 对删除、修改、支付、短信、邮件、批量导出接口生成 fuzz 计划。
