# /deep-l1

用途：使用 L1 快速低影响策略，对当前项目做接口、敏感信息和高价值入口的快速初筛。

执行规则：

1. 先读取 `CLAUDE.md`、当前项目声明和 `Competence/workflow-and-verification.md`；并读取 `memory/INDEX.md` 召回相关历史经验；复杂 / 多目标时读取或建立 `plan.md`，逐项推进。
2. 按 L1 策略执行：快速信息提取、围绕 `info-find` 命中上下文审计 JS / 小程序代码 / App 反编译代码、接口访问归类和 L1 默认测试项。
3. **上下文经济学**：L1 注重速度，大文件（> 500KB）/ 大目录（> 50 文件）/ 批量接口（> 20 个）优先 delegate，自己只接收结论（见 `Competence/workflow-and-verification.md` Delegate 机制）。
4. L1 默认测试项只包括未授权、信息泄露、SQL 注入闭合 / 排序测试、特有小字典参数 fuzz。
5. 默认测试项以外的漏洞类型，先和用户确认测试方案。
6. 只在允许范围内工作；危险接口禁止测试，只记录原因和位置。
7. 目标过多时分批处理，未完成范围必须标记为“部分测试”或“未完成”。
