# 微信小程序授权 code 报错泄露特长

## 目标

该特长只关注一种挖掘方向：微信小程序授权或登录流程中，前端将小程序生成的 `code` 发送给后端后，后端在异常处理时是否把敏感配置带入错误响应。

启用该特长后，当前阶段只做该方向。不要扩展到其他登录测试、账号测试、爆破、泛化接口 fuzz 或无关漏洞类型。

## 适用前提

- 已经掌握小程序授权或登录流程接口线索。
- 已经确认 `code` 参数的位置和传参方式。
- 有可用于复现的测试请求样例。

边界、范围和是否允许测试，由项目声明模块统一判断；本特长只描述该挖掘手法本身。

## 输入

优先从以下线索中定位目标接口：

- 小程序登录接口。
- 小程序获取手机号接口。
- 小程序授权接口。
- 前端或抓包中出现的 `code` 参数。
- 与小程序授权、登录、手机号换取相关的后端接口。

记录接口来源、请求方法、参数位置和 scope check 结果。

## 工作方式

1. 先确认接口属于微信小程序授权或登录流程。
2. 确认 `code` 参数位置：
   - GET / query 参数。
   - POST JSON 字段。
   - 表单字段。
3. 使用正常请求确认接口行为，只记录必要结果摘要。
4. 对 `code` 参数做最小化异常变形。
5. 观察响应中是否出现敏感配置或敏感字段。
6. code 变形后报错或返回异常，只能说明触发了异常响应；未出现敏感字段时，不得判断为漏洞。
7. 如果异常响应中已经出现敏感字段，立即停止该目标后续变形，只做脱敏记录，不继续扩大验证。

## code 变形思路

只对 `code` 参数做轻量异常输入，优先使用单个变形点：

- 在 `code` 中增加空格。
- 在 `code` 中增加百分号。
- 根据参数所在位置选择必要编码。

编码原则：

- GET / query 中使用 URL 编码。
- JSON body 中使用 Unicode 编码。
- 表单字段按实际请求编码方式处理。

不要对其他参数做无关 fuzz。

## 漏洞判断

code 变形后报错或返回异常，不等于漏洞。

只有异常响应中出现以下敏感内容时，才判断为确认泄露：

- `appid`
- `appsecret`
- `app_secret`
- `secret`
- `access_token`
- `https://api.weixin.qq.com/sns/jscode2session`
- 其他可识别的小程序敏感配置字段

若出现 `appsecret`、`secret`、`access_token` 等敏感值，立即停止该目标后续变形，证据必须脱敏，仅保留字段存在性说明或前后少量字符。

## 只做这些

- 定位小程序授权或登录 code 交换接口。
- 对 `code` 参数做一到两个最小化异常变形。
- 判断异常响应是否泄露 appid、appsecret、secret、access_token、jscode2session 或同类敏感配置 / 调用细节。
- 将结果写入当前项目的 `tests.md`、`findings.md` 或 `dangerous-untested.md`。

## 不做这些

- 不测试账号爆破、验证码绕过、密码喷洒。
- 不批量生成或批量尝试 code。
- 不测试真实用户手机号或真实用户数据。
- 不扩展到无关登录逻辑漏洞。
- 不对所有接口做通用 fuzz。
- 不保存完整 appsecret、Token、Cookie 或敏感响应包。

## 停止条件

出现以下情况立即停止并询问用户：

- code 来源不清或涉及真实用户。
- 响应出现 appid、appsecret、secret、access_token、jscode2session 或同类敏感配置 / 调用细节。
- 接口出现限流、验证码升级、封禁、异常变慢或服务错误。
