# 特长模板

> 这是模板，不是真实特长。真实特长应放在 `.claude/skills/specialty-skills/<特长名>/SKILL.md`。

## 特长定位

特长是某一种渗透测试思路或挖掘手法，不是普通基础 skill。

启用某个特长后，当前阶段只围绕该特长相关方向工作。除非用户退出特长模式，否则其他漏洞类型、信息收集方向、泛化测试思路一律不展开。

## 建议结构

```markdown
---
name: specialty-name
description: 用一句话说明这个特长适合什么渗透测试场景，以及它只关注哪类挖掘方向。
---

# 特长名称

## 目标

说明该特长要解决的渗透测试问题。

## 适用前提

- 该特长需要哪些前置信息。
- 该特长需要哪些输入材料。
- 该特长开始前应满足哪些技术条件。

边界、范围和是否允许测试，由项目声明模块统一判断；特长只描述手法本身。

## 输入

- 需要哪些线索。
- 需要哪些资产。
- 需要哪些接口或文件。

## 工作方式

- 如何从线索提出该特长相关的漏洞假设。
- 如何排序目标。
- 如何做最小化验证。
- 什么时候停止并询问用户。

## 只做这些

- 列出该特长允许关注的方向。

## 不做这些

- 列出该特长之外必须忽略的方向。
- 列出高风险或禁止行为。

## 输出

- 结果记录格式。
- 需要写入项目的文件。
```

## 编写原则

- 聚焦一个清晰挖掘思路，不写成大而全流程。
- 不要把基础信息提取、报告生成、项目声明管理写成特长。
- 不要在特长里重复项目声明模块已经负责的通用边界。
- 特长可以调用普通 skill，但普通 skill 本身不是特长。
