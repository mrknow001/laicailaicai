# 经验库管理

经验库让智能体越来越接近有经验的渗透测试人员，但不能变成敏感数据仓库。

## 可写入内容

- 某类系统常见接口命名。
- 某类认证或权限问题的排查思路。
- 某些框架、组件、技术栈与高价值路径的关联。
- 某些业务场景常见越权点。
- 某些测试方式容易误报或触发风控的原因。
- 不同 SRC 平台或企业项目的收录偏好，需脱敏。
- WAF / 风控的低影响观察和避坑方法。

## 禁止写入

- 真实用户数据。
- Token、Cookie、密钥、证书私钥。
- 未脱敏请求包和响应包。
- 可直接复现未公开漏洞的敏感目标细节。
- 客户源码、配置文件、数据库内容。
- 可用于未授权攻击的批量利用步骤。

## 文件分工

- `memory/experience.md`：通用渗透经验。
- `memory/make-mistakes.md`：误报、踩坑、错误方向、触发风控的教训。
- `memory/fingerprints.md`：指纹与漏洞方向关联。
- `memory/waf.md`：WAF / 风控观察。

## 写入格式

建议每条经验包含：

- 标题
- 场景
- 触发线索
- 判断方法
- 最小化验证思路
- 适用边界
- 不要做什么
- 来源：项目脱敏摘要或用户补充

写入前先脱敏，写入后确保不会泄露项目、用户或漏洞敏感信息。
