# 项目声明

项目声明是渗透测试智能体的规则源。任何目标访问、扫描、fuzz、PoC 验证、接口测试前，都必须先读取当前项目声明。

## 必填字段

- 项目名称
- 测试账号
- 允许域名
- 允许 IP / IP 段
- 禁止域名
- 禁止 IP / IP 段
- 漏洞收录要求
- 注意事项
- 补充说明

如果用户没有提供漏洞收录要求、注意事项或补充说明，使用 `templates/project.md` 中的默认内容。

## 范围判断

判断目标是否可测时按以下顺序执行：

1. 解析目标中的域名、IP、BaseURL。
2. 先检查是否命中禁止域名或禁止 IP。
3. 再检查是否命中允许域名或允许 IP。
4. 命中禁止范围时拒绝测试，即使同时命中允许主域。
5. 未命中允许范围或归属不清时，写入待确认线索并询问用户。

域名优先于 IP。允许主域不代表所有第三方跳转、CDN、云桶、认证中心、支付、短信、邮件服务都可测。

## 漏洞收录要求

漏洞收录要求用于决定测试优先级。智能体必须优先测试项目收录的漏洞类型，减少低价值动作。

默认优先关注：

- 未授权访问
- 越权访问
- 敏感信息泄露
- 认证绕过
- 严重业务逻辑漏洞
- 可利用 SQL 注入的存在性证明
- 文件上传 / 文件下载风险
- RCE / 命令执行存在性证明
- 直接影响账号安全、系统权限、核心数据的漏洞

默认低优先级：

- 普通 URL 跳转
- 普通反射 XSS
- 无敏感影响的信息泄露
- 低价值配置暴露
- 无法证明实际影响的轻微信息暴露

当项目只收高危或严重漏洞时，不要主动偏向 URL 跳转、普通 XSS 等低价值问题。

## 注意事项

注意事项是项目红线。默认禁止：

- 爆破、撞库、密码喷洒。
- 高并发扫描、DoS、影响稳定性的 fuzz。
- 删除、修改、支付、短信、邮件、批量导出接口测试。
- 未授权删除、越权删除、越权修改等破坏性验证。
- 批量遍历真实用户数据。
- 上传恶意代码、保留后门、读取敏感文件、下载源码。
- 提权、内网横向、持久化控制。

如果用户提供更严格的注意事项，必须采用更严格版本。

## 补充说明

补充说明用于项目级纠偏。通过 `/update-note` 追加后，后续所有工作都必须遵守。

补充说明适合写：

- 本项目不要再测某类低价值漏洞。
- 某类接口只记录不测试。
- fuzz 必须降速或禁止。
- 当前阶段集中看某类高价值漏洞。
- 某些误报方向不要重复尝试。

补充说明不能绕过全局安全红线。

## 输出

创建或更新项目声明后，输出：

- 项目名称
- 项目路径
- 允许范围摘要
- 禁止范围摘要
- 漏洞收录重点
- 注意事项摘要
- 当前补充说明
- 下一步建议
