# 字典生成

字典生成必须服务于明确的渗透测试目标，优先用于发现高价值接口、参数和业务入口。

## 允许生成

- 路径字典
- 参数名字典
- API 补全字典
- 备份文件名字典
- 业务关键字字典
- 排序字段字典
- 框架和组件路径字典

## 谨慎生成

用户名、密码类字典只能用于：

- 风险研判
- 报告说明
- 测试账号自查
- 用户明确提供的测试环境说明

禁止用于真实账号爆破、撞库、密码喷洒。

## 禁止生成

- 破坏性 payload 字典。
- 针对真实账号体系的爆破计划。
- 针对删除、修改、支付、短信、邮件、批量导出接口的 fuzz 计划。
- 大规模高并发请求计划。

## 生成依据

生成字典前先读取：

- 当前项目声明
- `leads.md`
- `interfaces.md`
- `memory/experience.md`
- `memory/fingerprints.md`
- `dicts/INDEX.md`

从以下线索提炼词根：

- 公司名称、项目名称、域名缩写
- 业务模块
- 接口路径
- 参数命名习惯
- 前端路由
- 错误提示
- 技术栈和框架指纹

## 输出要求

输出必须包含：

- 字典类型
- 适用目标
- 生成依据
- 风险等级
- 建议请求量
- 建议并发
- 熔断条件
- 字典内容或保存路径

默认建议低速、小字典、可停止。大型字典或深度 fuzz 只能在 L3 深度白名单策略下执行，并且必须明确白名单、字典规模、请求速度、并发限制、时间窗口和熔断条件。
