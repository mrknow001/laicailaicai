# 前端资产接口来源分析

Web、App、小程序不是三套独立渗透对象，而是三种前端资产呈现方式。它们的共同目标都是提取后端接口、参数、认证逻辑、业务流程和敏感配置线索。

真正进入渗透测试的是后端域名、IP、API BaseURL、接口、认证、权限、数据边界和业务逻辑。

## 核心原则

- 只做静态提取和只读分析。
- Web 可以通过浏览器、只读请求和 JS 审计提取线索；App 和小程序主要通过用户提供的包、源码、解包结果或分析材料提取线索。
- 提取到的域名、IP、BaseURL 必须回到项目声明中做 scope check。
- 第三方服务、CDN、云桶、认证中心、支付、短信、邮件等默认进入待确认线索，不直接测试。
- 不保存 Token、Cookie、密钥、证书私钥等敏感原文，只记录脱敏摘要和来源位置。
- 涉及删除、修改、支付、短信、邮件、批量导出、订单、真实用户数据的接口禁止测试；除非项目声明或补充说明明确允许具体接口、测试方式和停止条件。

## 统一工作流

1. 从前端资产中提取 URL、BaseURL、接口路径、参数、认证关键字、敏感配置摘要和业务关键字。
2. 围绕提取结果按需审计 JS、客户端代码、页面路由和配置文件，理解接口用途、参数来源、认证逻辑和业务流程。
3. 将有效线索写入当前项目的 `leads.md`、`interfaces.md`，必要时补充 `hypotheses.md`。
4. 对提取到的域名、IP、BaseURL 做 scope check。
5. 只把 scope check 通过的后端接口交给当前 L 档或特长继续渗透测试。

## Web / 静态文件

适合提取：

- API BaseURL
- 接口路径
- 参数名
- 路由
- 认证头名称
- 业务关键字
- 错误码和提示文案
- 静态敏感配置摘要

本地文件和目录的正则信息提取使用 `.claude/skills/info-find/SKILL.md`，规则和脚本由该 skill 自带。

## App

适合提取：

- API / BaseURL
- 认证逻辑线索
- 客户端权限判断线索
- 加固、混淆、证书和配置摘要
- 业务模块与接口关系

只分析用户提供的 APK、反编译目录或 App 分析材料。

## 小程序

适合提取：

- 页面路由
- API / BaseURL
- 请求参数
- 业务模块
- 登录态和角色线索
- 敏感配置摘要

只分析用户提供的小程序 appid、小程序包、源码目录或分析材料。appid 场景可按需使用 `.claude/skills/wxapkg-windows-unpack/SKILL.md` 先解包还原。

## 输出格式

每条线索建议记录：

- 来源类型：本地文件 / Web / App / 小程序 / 搜索 / 用户补充
- 来源位置：文件、URL、包名、页面路由
- 接口或 BaseURL
- 参数名
- 疑似功能
- 风险标签：只读查询、状态变更、敏感数据、认证相关、文件相关、支付/短信/邮件等
- scope check 状态：通过、禁止、待确认
- 漏洞假设
- 是否允许进入验证

输出写入当前项目的 `leads.md` 和 `interfaces.md`。
